# Neo UI Documentation

Neo UI is a Unity 6 clean-room rebuild of Doozy UI Manager 4. The authoritative spec for
architecture, principles, and workflows is **[CLAUDE.md](../../CLAUDE.md)** at the repo root —
every doc below defers to it if anything ever conflicts.

New to the package? Start with **[authoring-guide.md](./authoring-guide.md)** — a one-page mental
model of every surface (Hub, Setup, Design System, native authoring, Agent Bridge) with links out
to the deeper doc or code that owns each one.

## Decision tree: how should I build this UI?

- **I want to click a screen together in the Scene view** → `GameObject → Neo UI → …` +
  the scene-view overlay. See [authoring-guide.md § Build screens natively](./authoring-guide.md#build-screens-natively)
  and [native-authoring-testing-guide.md](./native-authoring-testing-guide.md) (test checklist,
  doubles as a walkthrough of every native-authoring surface).
- **I'm an agent (or scripting one) driving UI from JSON** → the Agent Bridge `sync` action.
  See [authoring-guide.md § Agents and the spec](./authoring-guide.md#agents-and-the-spec--the-sync-workflow),
  the field-by-field vocabulary in [spec-reference.md](./spec-reference.md), and the raw
  [neo-spec.schema.json](./neo-spec.schema.json).
- **I'm wiring generated UI to game code** (signals, data sources, cheats) →
  [developer-binding-guide.md](./developer-binding-guide.md).
- **I'm extending the package** (a new widget kind, theme token, animator role, flow node,
  shape effect, particle module) → there is no separate extensibility doc; every seam is
  cataloged in [CLAUDE.md § Extensibility seams](../../CLAUDE.md) with its registry/SO/virtual-hook
  entry point. [editor-ux-analysis.md](./editor-ux-analysis.md) covers the editor-kit
  (`Neo.EditorUI`) conventions for building matching inspectors/windows.
- **I'm authoring a NeoShape-based shader effect** → [neoshape-channel-layout.md](./neoshape-channel-layout.md).

## Full documentation map

**Getting started**
- [authoring-guide.md](./authoring-guide.md) — the mental model; start here.

**Building & authoring**
- [native-authoring-testing-guide.md](./native-authoring-testing-guide.md) — native (scene)
  authoring, walked through as a test checklist.
- [developer-binding-guide.md](./developer-binding-guide.md) — wiring generated/native UI to
  game code (signals, `UIData`, settings, cheats).

**Reference**
- [spec-reference.md](./spec-reference.md) — the JSON spec vocabulary; auto-generated by the
  `specReference` Agent Bridge action (`Tools → Neo UI → Advanced → Generate Spec Reference`) —
  never hand-edit the generated sections, regenerate instead.
- [neo-spec.schema.json](./neo-spec.schema.json) — the same schema, machine-readable.
- [neoshape-channel-layout.md](./neoshape-channel-layout.md) — `NeoShape` SDF vertex-channel
  layout for shader authors.
- [recipes/](./recipes/) — worked example specs (`confirm-dialog.json`, `game-hud.json`,
  `settings-screen.json`, `shop.json`).

**Developer / extending**
- Extensibility seams (registries, ScriptableObject discovery, virtual hooks) are cataloged in
  [CLAUDE.md](../../CLAUDE.md) — that list is the source of truth, not a doc under `Assets/docs/`.
- [editor-ux-analysis.md](./editor-ux-analysis.md) — editor UX rationale and the `Neo.EditorUI`
  field/control catalog, for anyone building a new inspector or window.


**Glossary**
- [GLOSSARY.md](./GLOSSARY.md) — one-page definitions of signal, registry, seam, flow graph,
  theme token, shape style, widget preset, animation preset, and the rest of the package's jargon.

## What got pruned (2026-07-11 audit)

The following planning docs were deleted because the work they describe has already shipped, or
because they described the "Composer" window, which was retired in favor of native authoring — see
CLAUDE.md and `native-authoring-testing-guide.md § 5` for the current story. Git history has the
originals if you need the rationale behind a shipped decision:

- `composer-authoring-overhaul/` (9-part Composer redesign — the whole window was retired, not built)
- `composer-catalog-unification-plan.md`, `extensibility-seams-master-plan.md`,
  `extensibility-seam-{element-kinds,flow-triggers,theme-bundles,validation-rules,widget-attributes}-plan.md`,
  `widget-presets-plan.md`, `settings-cheats-menu-plan.md`, `design-system-cohesion-plan.md`
  (all shipped — the registries/tabs/fixes they proposed exist in code today)
- `human-workflow-plans/` (all four plans shipped or retired; plan 2 was the Composer)
