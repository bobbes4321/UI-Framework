Settings & cheat menus

Describe your settings screen as a catalog — groups, items, defaults — and the framework builds the rows, persists the values, and broadcasts every change as a signal.

How it fits together

CatalogSettingsCatalog / CheatCatalog asset
→
MenuPresenterbuilds rows from the widget library
→
Row + binderwidget wired to the value
→
UserSettingsServiceget / set / persist / signal

A MenuCatalog ScriptableObject declares groups (rendered as tabs) and items — each item is a category/name id, a row kind, a label, and kind-specific fields like min/max or options. At startup, SettingsMenu / CheatMenu (both thin specializations of MenuPresenter) clone one row prefab per item from the MenuWidgetLibrary, and a MenuControlBinder on each row wires the widget to the value. You never hand-build a settings screen row by row.

Extensibility seam

Row kinds live in the NeoMenuItemKinds registry — register your own kind (say, a color picker row) and it becomes available to catalogs and specs like the built-ins.

Values: reading, writing, persisting

UserSettingsService is the single door for every menu value. A read resolves in order: a live code binding if the game registered one, then the store (PlayerPrefs by default, keys namespaced NeoUI.Settings.* — or any IUserSettingsStore you inject), then the catalog's default. Hook a setting to real game state like this:

// route "Audio/Master" through the mixer instead of PlayerPrefs alone
UserSettingsService.Bind<float>("Audio", "Master",
    getter: () => GetMixerVolume(),
    setter: v  => SetMixerVolume(v),
    persist: true);  // also write the store, so the value survives restarts

// or just react to changes anywhere
Signals.On("Settings", "Video/VSync", ApplyVSync);

Every committed change fires a signal on the Settings (or Cheat) category. Sliders can additionally emit preview signals while dragging (emitOnDrag) on a separate Settings.Preview stream, so you can live-preview volume without persisting every intermediate value — the commit happens on release.

Cheat menus

A cheat menu is the same machinery pointed at a different signal category (Cheat instead of Settings), so cheats never collide with real settings. On top of that, CheatMenu adds favourites: enable them on the catalog and testers can star their most-used cheats into a quick-access group (persisted per user via the same service). Typical cheat rows are buttons (fire an action), toggles (god mode), and steppers (add coins).

Input rebinding

A rebind row binds to a New Input System action by path (e.g. Gameplay/Jump) plus a binding index for composites. The row shows the current binding's display string ("Space", "Left Mouse"); clicking it starts an interactive "press a key…" rebind through InputRebindService, which saves overrides as JSON per action asset and restores them automatically on startup. A reset removes the override and restores the default binding.

In a spec

The top-level settings and cheats sections declare whole catalogs; place a settings/cheats element in a view to embed the generated menu:

"settings": [
  { "id": "Settings/Game",
    "groups": ["Audio", "Video", "Controls"], "start": "Audio",
    "items": [
      { "slider": { "id": "Audio/Master", "group": "Audio",
                    "label": "Master Volume",
                    "min": 0, "max": 1, "value": 0.8 } },
      { "toggle": { "id": "Video/VSync", "group": "Video",
                    "label": "VSync", "value": true } },
      { "rebind": { "id": "Controls/Jump", "group": "Controls",
                    "label": "Jump", "action": "Gameplay/Jump",
                    "bindingIndex": 0 } }
    ] }
]

Useful item fields beyond the basics: tooltip (helper text), persisted: false (the game owns the value — nothing written to the store), wholeNumbers on sliders (integer values), options on dropdowns, and step on steppers. The settings-menu showcase demos all of it.

Styling the rows

The row prefabs come from the MenuWidgetLibrary asset, installed by Setup → Create or Repair Menu Widget Library — eight row prefabs plus the category tab and panel used for group navigation. Restyle or replace those prefabs (they're ordinary Neo widgets) and every menu built from the library picks up the change. Validate confirms that control bindings resolve to catalog entries and that rebind rows name a real action.
