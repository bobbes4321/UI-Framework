Flow graphs

Screen navigation as a state machine: nodes show views, edges fire on triggers, history enables Back.

Extensibility seam

Register your own trigger kinds via NeoTriggerKinds.Register — implement INeoTriggerKind (Connect, Matches), optionally ITriggerKindIdDatabase to get an id picker in the editor. Flow node kinds themselves are registry-driven too (FlowNodeKinds).

What a UINode does

On enter, a UINode hides its hideViews list, shows its showViews list (with transition choreography if the edge named one), then listens for its output-edge triggers. On exit, it optionally hides what it showed (hideShownViewsOnExit) — and views shared with the next node carry over untouched, so navigating between two screens that both show the HUD never hide-and-reshow-flashes it.

Authoring in the Flow Graph Editor

Creating nodes: right-click the canvas for a search-as-you-type node picker, or drag off an output port into empty space — the picker appears and the new node is auto-wired to that port. Double-clicking a FlowGraph asset also opens the window.

Configuring edges: each edge carries its target node (by name), its trigger, an allowsBack flag (may GoBack() return across it? default yes), an optional port label, a weight (for weighted-random selection), and an optional named transition — empty means the project default.

Toolbar: Validate, Arrange (auto-layout), and Frame All / Frame Selection. In play mode the window pulses live: the active node highlights in place, traversed edges flash, and a breadcrumb strip tracks the controller's history.

Running a flow: FlowController

Add a FlowController to a scene object and assign a graph. It clones the asset at start, so runtime state never touches the asset. Key inspector options:

OptionMeaning
flowTypeGlobal (findable from anywhere via FlowController.GetGlobalController()) or Local (a self-contained sub-flow, e.g. inside one widget).
onEnableBehaviour / onDisableBehaviourWhat enabling/disabling the component does: DoNothing · StartFlow · StopFlow · PauseFlow · ResumeFlow. Default: start on enable, stop on disable.
dontDestroyOnSceneChangeKeep a global flow alive across scene loads.
goBackOnBackButton / autoCreateBackInputNavigate back when the back-button signal fires, and auto-add the hardware listener if the scene lacks one.

At runtime you get StartFlow / StopFlow / PauseFlow / ResumeFlow (pause suspends timers), GoBack() (pops the history stack, only across allows-back edges), SetActiveNodeByName() for direct jumps, and read access to activeNode, previousNode, and history.

Flow in a spec

The spec's flow section describes the same graph as data — here's a node from the game-ui showcase:

"flow": {
  "name": "GameUI", "start": "MainMenu",
  "nodes": [
    { "name": "Playing", "view": "Game/HUD",
      "next": [
        { "to": "Inventory", "on": { "button": "HUD/Inventory" } },
        { "to": "Victory",   "on": { "button": "HUD/Finish" },
          "transition": "Modal/ZoomIn" },
        { "to": "MainMenu",  "on": { "back": true } }
      ] }
  ] }

Trigger forms: {"button": "Cat/Id"} · {"signal": "Cat/Name"} · {"toggleOn"/"toggleOff": "Cat/Id"} · {"viewShown"/"viewHidden": "Cat/Id"} · {"back": true} · {"timer": 3.5} (seconds in the node).

View transitions & shared elements

ViewTransitionAsset makes a navigation cut a first-class, named thing: it can override the outgoing view's hide and the incoming view's show, offset their timelines, cascade children, and fly NeoSharedElement-marked widgets across the cut (shared-element transitions). Edges reference transitions by name; NeoUISettings holds the project default. Eight ship with the transition library:

Fade/CrossFade/ThroughBlack Push/SlideLeftPush/SlideRightPush/SlideUpPush/SlideDown Modal/ZoomInModal/SheetUp

Like animation presets, transition assets are discovered automatically — add your own next to the shipped ones. The transitions showcase demos the library.

Signals

Named event streams that decouple senders from receivers — the framework's connective tissue.

// Send — creates the stream on first use
Signals.Send("Audio", "Muted");

// Subscribe — parameterless or with the full Signal (sender, source, timestamp, message)
Signals.On("Audio", "Muted", () => audioSource.mute = true);
Signals.On("Game", "Over", (Signal s) => Debug.Log(s.message));
TypeRole
SignalStreamA named channel (Category/Name): send, connect/disconnect receivers.
Signal / MetaSignal<T>The message — payload-less by default; MetaSignal<T> carries a typed payload.
SignalsStatic service: get-or-create streams, Send, On/Off subscription.
SignalReceiverSerializable receiver embeddable in any component — pick a stream in the inspector, hook a UnityEvent or callback.
SignalSenderComponent / SignalReceiverComponentDrop-on components: send on enable/disable/destroy, or bind a stream to a UnityEvent — no code.
SignalLoggerLogs every signal on a stream — first stop when debugging "why didn't X react".

Widgets emit signals natively: a toggle can publish Audio/Muted, a button behavior can send on click, flow edges listen for them, particle emitters can burst on them, and effects can bind signal payloads to parameters (NeoSignalParamBinding). Prefer signals over direct UnityEvent references whenever the sender and receiver live in different prefabs.

Back navigation

One unified sink for "go back", no matter where the press came from.

BackButton.Fire() is the single entry point, publishing on the Input/BackButton stream. Three sources feed it:

Hardware

BackButtonInput converts Escape and gamepad Cancel into Fire().

Named buttons

Any UIButton whose name matches the configured back-button name fires it automatically.

Code

Call BackButton.Fire() directly; ForceFire() bypasses the cooldown.

Flow edges with the back trigger consume the press and pop the flow history. Disable()/Enable() stack, so nested "back is temporarily forbidden" scopes compose correctly, and a cooldown (configurable in settings) prevents double-fires.

Invariant

Explicit back wiring consumes the press — a screen with a wired back edge never also triggers the global fallback, so you never double-navigate. Covered by BackNavigationTests.
