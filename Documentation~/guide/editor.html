The ID Database Manager

Since everything is addressed by Category/Name, the id registries are the project's backbone — and this window is where you keep them healthy. Eight databases ship (Views, Buttons, Toggles, Sliders, Dropdowns, Panels, Tags, signal Streams; add your own via INeoIdDatabaseProvider), browsed in three panes: database → categories → names. Three tabs:

Browse & Edit

Add, rename, and delete categories and names inline, with a substring filter and a row filter (All / Orphans / Dangling).

Search

One global substring search across every database at once — "where is anything called Pause?"

Usage / Orphans

Cross-references ids against actual use: orphans are registered but unused (⚠ badges), dangling ids are used but unregistered — with quick-fix add/delete buttons.

Renames can optionally rewrite references: with the toggle on, renaming an id also updates matching references in your spec files, after a preview dialog showing exactly which files and how many references change. Prefabs and scenes are not rewritten directly — run a sync/regenerate afterward to materialize the rename. Every inspector id picker links back here too: the quick-add button registers new ids without leaving the inspector, and the jump button opens this window focused on the exact id.

Editor performance

By design there is no animated editor chrome, no editor-tick subscriptions, no reflection scans, and exactly one settings asset. Inspectors cache their GUIStyles and SerializedObjects. If you extend the editor, follow suit.

Validation

Three buckets: hard contracts that can fail CI, design lints that advise, and an interactivity lint that catches dead UI.

Hard contracts

Theme coverage (every variant × every token) · theme bindings resolve · popup database completeness · preset database completeness · generated views registered · flow graph syntax · flow cross-references (views/buttons exist) · menu bindings resolve.

Design lint (soft)

WCAG contrast on text/surface pairs · raw font sizes where a TextStyle exists · spacing off the 4/8/12/16/24/32/48/64 scale · missing icon glyphs · tap targets under 44px · extreme sprite stretch · fixed-size elements under force-expanding layouts · Tier-2 effect batching cost · particle emitter cost.

Interactivity lint (soft)

Dead buttons (no flow trigger, signal, popup, view command, or event) · dead tabs (no container, no flow trigger).

Extensibility seam

Add project-specific rules by implementing INeoValidationRule and calling NeoValidationRules.Register, choosing which bucket (Hard / Design / Interactivity) the rule runs in.
