Containers & views

Containers are the things that show and hide. Everything visible lives inside one.

ComponentWhat it is
UIContainerBase visibility container: show/hide with transitions, animator registration, interruption handling, auto-hide and auto-select behaviors.
UIViewA screen. Addressed by Category/Name, controllable from anywhere via static API, navigated by flow graphs.
UIPanelTab-content container, kept in sync with a UITab's selection; optional fade-in on enable.
UIPopupModal/popup looked up by name from PopupDatabase, configured with a fluent API, queued FIFO (one visible per queue).
UITooltipTooltip with follow/anchor positioning and hover triggering; anchors resolve via named UITag targets.
UIOverlayMarks children as z-stacked (overlay layout) rather than layout-group stacked.
UIResponsiveRootResponsive layout root managing breakpoint-aware canvas scaling.
UIResponsiveGridColumnsGrid whose column count adapts to the active screen breakpoint.

Controlling views from code

// Show / hide by address — from anywhere, no references needed
UIView.Show("Menu", "Main");
UIView.Hide("Menu", "Settings", instant: true);
UIView.ShowCategory("HUD");      // every view in a category
UIView.HideAllViews();

// Popups: fetch by name, configure fluently, then show
UIPopup.Get("Confirm")
       .SetTexts("Delete save?", "This cannot be undone.")
       .SetEvents(OnConfirm, OnCancel)
       .SetQueue("dialogs")
       .Show();

Every show/hide accepts an instant flag and an optional UIAnimation override. Containers handle interruption gracefully — hiding mid-show reverses from the current pose rather than snapping.

Interactive widgets

All widgets carry a Category/Name id, emit signals, and animate their selection states through the same animator model.

WidgetWhat it does
UIButtonButton with eleven behavior triggers (below), per-trigger cooldowns, and signal emission per behavior.
UIToggleOn/off control with group exclusivity, value-changed events, and domain signal publishing (e.g. emit Audio/Muted).
UISliderSlider distinguishing continuous (while dragging) from committed (on release) values, with min/max/increment events.
UIDropdownSingle-choice dropdown (extends TMP_Dropdown) with an id and a selection-changed signal.
UITabToggle bound bidirectionally to a target UIContainer — selecting shows the container, and the container's state mirrors back.
UIStepperPlus/minus value control with step size, min/max, and reach-limit events.
UICounterRolling number label that tweens its displayed value toward a target — score counters, coin totals.
UIBadge / UITagBadge labels, and named RectTransform position targets (for popup/tooltip anchoring).
UIScrollbarScrollbar with selection-state animator integration.
UIRebindControlUI for runtime input rebinding, backed by InputRebindService.

Button behavior triggers

Each UIButton hosts a list of UIActionBehaviours — one per trigger — each firing a UnityEvent, an optional signal, and honoring an optional cooldown:

ClickDoubleClickLongClickRightClick PointerEnterPointerExitPointerDownPointerUp SelectedDeselectedSubmit

Selection states

Widgets built on UISelectable expose five states — Normal · Highlighted · Pressed · Selected · Disabled — consumed by pluggable animators (UISelectableColorAnimator, UISelectableUIAnimator) via SelectionStateRelay. Spec-generated buttons come in variants primary secondary danger light ghost and sizes sm md lg.

Invariant

Dead-interaction lint. A button or tab with no flow trigger, signal, popup, view command, or event is flagged by Validate — a control that looks alive but does nothing never ships silently.
